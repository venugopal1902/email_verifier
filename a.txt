This structure provides the necessary components:

1.  **Isolation (SQL):** The `core/db_routers.py` handles the logic to direct Main DB models (`accounts` app) to the `default` (PostgreSQL) database and Account DB models (`files` app) to a dynamic, per-account database (required for FR07).
2.  **Concurrency (Redis/Celery):** `docker-compose.yml` sets up Redis as the broker and backend, and a dedicated `celery` worker service to process the heavy `process_verification_pipeline` task asynchronously (NFR05).
3.  **Core Logic (Python/Django):** `accounts/models.py` and `files/models.py` implement the PRD's schema, and `files/tasks.py` simulates the multi-stage verification and credit deduction (FR31, FR32).
4.  **UI (HTML/JS/CSS):** `dashboard.html` provides a basic, responsive interface to test the file upload API flow.
